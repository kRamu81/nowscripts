# Cloud Computing Basics

Welcome to Lesson 1 of the Administration course.

## Introduction to Cloud Computing

Cloud computing is one of the most important technologies used today. It allows people and businesses to access computing resources such as servers, storage, databases, and software over the internet instead of owning and managing physical hardware.

In the past, companies had to buy expensive computers, servers, and networking equipment before they could build websites or run applications. This required a large investment, regular maintenance, and dedicated IT teams.

Cloud computing changed this approach. Instead of purchasing hardware, users can rent the resources they need from cloud service providers. This makes it easier, faster, and more affordable to build and manage applications.

Cloud computing helps individuals and organizations:

- Access computing resources anytime through the internet.
- Reduce the cost of buying and maintaining physical hardware.
- Increase or decrease resources whenever needed.
- Launch applications quickly without waiting for new hardware.
- Work from anywhere using different devices.

**Example:** Imagine you want to start an online shopping website. Without cloud computing, you would need to buy servers, install software, set up networking, and manage everything yourself. With cloud computing, you can simply rent the required resources from a cloud provider, launch your website in a short time, and increase server capacity automatically during busy shopping seasons.

## What is Cloud Computing

Cloud computing is a technology that allows users to access computing resources such as servers, storage, databases, software, and networking over the internet instead of using their own physical hardware.

Instead of buying and maintaining expensive equipment, users can rent the resources they need from cloud service providers. This makes cloud computing affordable, flexible, and easy to use.

Cloud providers own and manage large data centers that contain powerful servers and storage systems. When you use a cloud service:

1. You connect to the service through the internet.
2. Your request is sent to the cloud provider's servers.
3. The provider processes the request and sends the results back to your device.
4. You pay only for the resources you use (depending on the service).

Cloud computing offers several important features:

- **On-demand access** – Resources are available whenever you need them.
- **Scalability** – Increase or decrease resources based on demand.
- **Broad network access** – Use cloud services from laptops, tablets, and smartphones.
- **Resource sharing** – Multiple users share cloud resources securely.
- **Pay-as-you-go pricing** – Pay only for what you use.

You probably use cloud computing every day without realizing it. Common examples include Gmail, Google Drive, Google Docs, Microsoft 365, Netflix, and Dropbox.

## Private and Public Cloud

Cloud services can be deployed in different ways based on an organization's needs. The two most common deployment models are Private Cloud and Public Cloud.

A **private cloud** is a cloud environment used by only one organization. The infrastructure is dedicated to a single business, providing greater control, security, and customization. Private clouds are commonly used by organizations that handle sensitive information — for example, a hospital storing patient records to keep medical information secure.

A **public cloud** is owned and managed by a cloud service provider and its services are available to anyone over the internet. Users pay only for the resources they use without managing the underlying hardware — for example, a small business hosting its website using Amazon Web Services (AWS) instead of purchasing its own servers.

| Feature     | Private Cloud           | Public Cloud                    |
|-------------|-------------------------|---------------------------------|
| Ownership   | Single organization     | Cloud provider                  |
| Cost        | Higher                  | Lower                           |
| Security    | Higher                  | Good, but shared infrastructure |
| Scalability | Moderate                | Very High                       |
| Best For    | Sensitive business data | Websites, apps, startups        |

Choose a **private cloud** when your organization handles confidential data, needs greater control over infrastructure, or when industry regulations require higher security. Choose a **public cloud** when you want a low-cost solution, need resources quickly, or your application must scale easily.

## What is the Difference Between SAAS and PAAS

Cloud providers offer different service models. Two popular models are Software as a Service (SaaS) and Platform as a Service (PaaS).

**Software as a Service (SaaS)** provides ready-to-use software over the internet. Users simply log in and start using the application without installing or maintaining it. Examples include Google Docs, Gmail, Microsoft 365, and Zoom.

**Platform as a Service (PaaS)** provides developers with a platform to build, test, and deploy applications. The cloud provider manages the servers and infrastructure, allowing developers to focus on writing code. Examples include Google App Engine, Heroku, and AWS Elastic Beanstalk.

| Feature           | SaaS                       | PaaS                      |
|-------------------|----------------------------|---------------------------|
| Purpose           | Use software               | Build applications        |
| Users             | End users                  | Developers                |
| Server Management | Managed by provider        | Managed by provider       |
| Coding Required   | No                         | Yes                       |
| Examples          | Google Docs, Microsoft 365 | Heroku, Google App Engine |

Choose **SaaS** when you need ready-made software, want quick access with no installation, or do not want to manage updates. Choose **PaaS** when you are developing applications, want to focus on coding instead of managing servers, or need a platform for testing and deployment.

## What are the Benefits of Cloud Computing?

Cloud computing offers many advantages for both individuals and businesses.

**Cost Savings** – Cloud computing reduces the need to buy expensive hardware and maintain data centers. A startup, for example, can launch its website without purchasing servers.

**Scalability** – Resources can be increased or reduced whenever demand changes. An online shopping website can add more servers during a festival sale and reduce them afterward.

**Flexibility** – Users can access cloud services from almost anywhere using an internet connection.

**Reliability** – Cloud providers maintain backup systems to help keep services available.

**Security** – Cloud providers offer security features such as encryption, access control, and Multi-Factor Authentication (MFA).

**Faster Deployment** – Applications and services can be launched much faster than setting up physical infrastructure.

**Accessibility** – Users can access files and applications from laptops, tablets, and smartphones.

In summary, cloud computing provides lower costs, easy scalability, better flexibility, improved reliability, enhanced security, faster deployment, and access from anywhere.
