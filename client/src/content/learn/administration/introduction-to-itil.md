# Introduction to ITIL

Welcome to Lesson 2 of the Administration course.

## What is ITIL

ITIL stands for **Information Technology Infrastructure Library**. It is a set of best practices that helps organizations manage their IT services in a simple and organized way.

Think of ITIL as a rulebook that tells IT teams how to do their job properly — from fixing problems to delivering new services.

> **Example:** When you raise a ticket in ServiceNow for a laptop issue, the process of logging, assigning, and resolving that ticket follows ITIL practices.

## Introduction to ITIL Foundation

ITIL Foundation is the basic level of ITIL. It covers the key ideas and vocabulary used in IT service management.

It helps beginners understand how IT services are planned, delivered, and improved. In ServiceNow, most processes like Incident Management, Change Management, and Problem Management are built on ITIL Foundation concepts.

> **Example:** When a user cannot log in and raises a ticket, the ServiceNow Incident module handles it using ITIL Foundation guidelines.

## ITIL Versions

ITIL has been updated over the years to stay relevant with modern IT needs.

| Version | Year | Key Focus |
|---------|------|-----------|
| ITIL v1 | 1989 | Basic IT processes for UK government |
| ITIL v2 | 2001 | Service Support and Delivery |
| ITIL v3 | 2007 | Service Lifecycle approach |
| ITIL 4  | 2019 | Agile, DevOps, and modern IT practices |

ServiceNow is aligned with **ITIL 4**, which focuses on flexibility, collaboration, and continuous improvement.

## Benefits of ITIL Standards

Using ITIL in ServiceNow brings clear advantages to IT teams and businesses.

- **Fewer Disruptions** — Incidents are handled faster with a proper process.
- **Better Service Quality** — IT teams follow a standard way of working.
- **Clear Roles** — Everyone knows what they are responsible for.
- **Faster Problem Solving** — Root causes are found and fixed properly.
- **Cost Savings** — Efficient processes reduce wasted time and effort.

> **Example:** A company using ServiceNow with ITIL reduces repeated incidents because problems are properly investigated and resolved at the root level.

## ITIL Approaches

ITIL uses a service-focused approach to manage IT. The key idea is the **Service Value System (SVS)** in ITIL 4, which ensures every IT activity adds value to the business.

The main approaches are:

- **Plan** — Understand what the business needs.
- **Improve** — Continuously make services better.
- **Engage** — Work closely with users and stakeholders.
- **Design & Transition** — Build and launch new services properly.
- **Obtain & Build** — Get the right tools and resources.
- **Deliver & Support** — Run services and fix issues.

In ServiceNow, these approaches map directly to modules like Change Management, Service Catalog, and Incident Management.

## Roles and Responsibility of IT Help Desk

The IT Help Desk is the first point of contact when a user has a problem. In ServiceNow, the Help Desk operates using ITIL roles.

| Role | Responsibility |
|------|----------------|
| **L1 Agent** | Receives and logs tickets, solves basic issues |
| **L2 Agent** | Handles complex issues escalated from L1 |
| **L3 / Specialist** | Solves deep technical problems |
| **Incident Manager** | Oversees the full incident process |
| **Problem Manager** | Finds root causes of repeated incidents |

> **Example:** A user raises a ticket in ServiceNow because their email is not working. The L1 agent picks it up, tries basic fixes, and escalates to L2 if needed.

## Modules in ITIL

ITIL is organized into key modules, each managing a specific area of IT service. ServiceNow has built-in modules for each of these.

| ITIL Module | What it Does | ServiceNow Module |
|---|---|---|
| **Incident Management** | Fix disruptions quickly | Incident |
| **Problem Management** | Find and fix root causes | Problem |
| **Change Management** | Manage changes safely | Change |
| **Service Request** | Handle user requests | Service Catalog |
| **Knowledge Management** | Share solutions and guides | Knowledge Base |
| **Configuration Management** | Track IT assets and relationships | CMDB |

> **Example:** When a server goes down, Incident Management logs and resolves the outage. Problem Management then investigates why it happened to prevent it from recurring.
